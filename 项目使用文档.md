# 视频会议管理系统 - 项目使用文档

## 一、项目概述

### 1.1 系统简介

视频会议管理系统是一个基于 WebRTC 技术的实时视频会议与直播平台，提供视频会议、在线直播、实时表决、权限管理等功能。系统采用前后端分离架构，支持多人实时音视频通信、桌面共享、在线投票等协作功能。

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| **后端框架** | Python + FastAPI (支持异步和 WebSocket) |
| **前端框架** | Vue 3 + Vite + Element Plus |
| **数据库** | SQLite (可无缝迁移至 MySQL) + SQLAlchemy ORM |
| **实时通信** | WebSocket + WebRTC (P2P Mesh 网络拓扑) |
| **鉴权机制** | JWT (JSON Web Token) |
| **部署方式** | Nginx 反向代理 + ngrok 内网穿透 (可选) |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      表现层 (Vue 3)                          │
│  UI渲染、用户交互、音视频流播放展示、WebRTC媒体处理           │
└─────────────────────────────────────────────────────────────┘
                              ↓ HTTP/WebSocket
┌─────────────────────────────────────────────────────────────┐
│               网关/路由层 (FastAPI Router)                   │
│  RESTful API、WebSocket长连接、请求路由分发                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              业务逻辑层 (Python Services)                    │
│  会议预约、权限校验、表决统计、WebSocket连接管理              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              数据层 (SQLAlchemy + SQLite)                    │
│  用户、会议、直播、表决、模板等数据持久化                     │
└─────────────────────────────────────────────────────────────┘
```

## 三、快速启动

### 3.1 启动方式

项目提供了统一的启动脚本 `start.sh`，支持多种启动模式：

```bash
# 同时启动前后端 (默认，推荐开发模式)
./start.sh

# 生产模式: Nginx + Ngrok 代理部署
./start.sh prod

# 仅启动后端 API 服务
./start.sh backend

# 仅启动前端 Vite 开发服务
./start.sh frontend

# 仅构建前端生产版本
./start.sh build

# 显示帮助
./start.sh help
```

### 3.2 端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8001 | FastAPI 服务 |
| 前端开发 | 5173 | Vite 开发服务器 |
| 生产入口 | 80 | Nginx 反向代理 |

### 3.3 访问地址

- **开发模式**:
  - 前端: http://127.0.0.1:5173
  - 后端: http://127.0.0.1:8001
  - API文档: http://127.0.0.1:8001/docs

- **生产模式**:
  - 通过 Nginx 80 端口统一访问
  - 可通过 ngrok 获取 HTTPS 公网地址

## 四、功能模块详解

### 4.1 用户权限模块

#### 4.1.1 角色定义

| 角色 | 权限范围 |
|------|----------|
| **admin (管理员)** | 全部权限：用户管理、会议模板管理、创建会议/直播、参与会议 |
| **host (主持人)** | 创建会议、发起直播、发起表决、参与会议 |
| **user (普通用户)** | 加入会议、观看直播、参与表决 |

#### 4.1.2 前端视图

| 路由 | 组件 | 功能 |
|------|------|------|
| `/login` | LoginView.vue | 用户登录 |
| `/register` | RegisterView.vue | 用户注册 |
| `/admin/users` | UserManageView.vue | 用户管理 (仅管理员) |

---

### 4.2 视频会议模块

#### 4.2.1 功能特性

- **会议预约**: 创建会议、设置主题和时间
- **会议室**: 实时音视频通信、桌面共享
- **会议状态**: scheduled (待开始) / ongoing (进行中) / ended (已结束)
- **会议模板**: 快速创建预设配置的会议

#### 4.2.2 前端视图

| 路由 | 组件 | 功能 |
|------|------|------|
| `/meetings` | MeetingListView.vue | 会议列表、创建会议 |
| `/meetings/:id` | MeetingRoomView.vue | 会议室控制台 |

#### 4.2.3 会议室功能

- **本地设备控制**: 打开/关闭摄像头和麦克风
- **桌面共享**: 共享屏幕内容
- **远端流管理**: 接收其他参会者的音视频流
- **WebRTC信令**: 通过 WebSocket 交换 SDP 和 ICE Candidate

---

### 4.3 直播模块

#### 4.3.1 功能特性

- **开启直播**: 创建直播间，生成唯一房间号
- **主播端**: 推送本地音视频流
- **观众端**: 接收主播画面（仅拉流模式）
- **直播回看**: 播放录制的直播视频

#### 4.3.2 前端视图

| 路由 | 组件 | 功能 |
|------|------|------|
| `/live` | LiveListView.vue | 直播列表、创建直播 |
| `/live/:id` | LiveRoomView.vue | 直播间控制台 |

---

### 4.4 在线表决模块

#### 4.4.1 功能特性

- **发起表决**: 主持人/管理员可发起表决，设置主题和选项
- **参与投票**: 所有参会者可提交投票
- **实时统计**: 通过 WebSocket 实时推送投票结果
- **结果展示**: 显示各选项的投票人数和比例

#### 4.4.2 前端组件

| 组件 | 功能 |
|------|------|
| VotePanel.vue | 表决面板，显示表决主题、选项和结果 |

---

### 4.5 后台管理模块

#### 4.5.1 功能特性

- **用户管理**: 查看用户列表、调整用户角色
- **会议模板管理**: 创建/编辑/删除会议模板

#### 4.5.2 前端视图

| 路由 | 组件 | 功能 |

## 五、后端 API 接口文档

### 5.1 认证接口 (Auth)

#### POST `/api/auth/register`
用户注册

**请求体**:
```json
{
  "username": "string (3-50字符)",
  "password": "string (6-128字符)"
}
```

**响应**:
```json
{
  "access_token": "string",
  "token_type": "bearer",
  "user": {
    "id": "integer",
    "username": "string",
    "role": "admin|host|user",
    "created_at": "datetime"
  }
}
```

**说明**: 第一个注册的用户自动成为管理员

---

#### POST `/api/auth/login`
用户登录

**请求体**:
```json
{
  "username": "string",
  "password": "string"
}
```

**响应**: 同注册接口

---

获取当前用户信息

**请求头**: `Authorization: Bearer <token>`

### 5.7 WebSocket 接口

#### WebSocket `/ws/meetings/{meeting_id}`
会议室信令通道

**消息类型**:

| 类型 | 方向 | 说明 |
|------|------|------|
| `join` | 客户端 → 服务端 | 加入房间，携带 `from` 字段 |
| `offer` | 客户端 → 服务端 | WebRTC Offer SDP |
| `answer` | 客户端 → 服务端 | WebRTC Answer SDP |
| `ice-candidate` | 客户端 → 服务端 | ICE Candidate |
| `vote-started` | 服务端 → 客户端 | 新表决发起通知 |
| `vote-result` | 服务端 → 客户端 | 投票结果实时更新 |

---

#### WebSocket `/ws/live/{live_id}`
直播间信令通道

**消息类型**: 同会议室，用于主播与观众的 WebRTC 连接建立

## 七、前端组件架构

### 7.1 状态管理 (Pinia)

#### AuthStore (auth.ts)
```typescript
// 状态
token: string           // JWT Token
user: UserProfile|null  // 当前用户信息
role: Role|''           // 用户角色

// 方法
restore()               // 恢复登录状态
loginAction()           // 登录
registerAction()        // 注册
logout()                // 登出
```

---

### 7.2 路由守卫

```typescript
// 路由元信息
meta: {
  public: true,         // 公开页面(无需登录)
  roles: ['admin']      // 需要的角色权限
}

// 路由守卫逻辑
1. public 页面 → 直接进入
2. 无 token → 跳转登录
3. 无用户信息 → 恢复登录状态
4. 角色不匹配 → 跳转会议列表
```

---

### 7.3 WebSocket 客户端 (WsClient)

```typescript
class WsClient {
  connect(url, onMessage)  // 建立连接
  send(payload)            // 发送消息(支持队列)
  close()                  // 关闭连接
}
```

**特性**: 支持消息队列，连接建立前发送的消息会被暂存，连接后自动发送

## 十、常见问题

### Q1: 摄像头/麦克风无法访问?

**原因**: WebRTC 需要 HTTPS 或 localhost 环境

**解决方案**:
- 本地开发使用 `http://127.0.0.1:5173`
- 生产环境使用 ngrok 获取 HTTPS 地址

---

### Q2: WebSocket 连接失败?

**检查项**:
1. 后端服务是否正常运行
2. Nginx WebSocket 代理配置是否正确
3. 浏览器是否支持 WebSocket

---

### Q3: 如何迁移到 MySQL?

**步骤**:
1. 修改 `config.py` 中的 `database_url`
2. 安装 MySQL驱动: `pip install mysqlclient`
3. 创建数据库
4. 重新运行应用(自动创建表结构)
