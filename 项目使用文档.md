# 视频会议管理系统 - 项目使用文档

## 一、项目概述

### 1.1 系统简介

视频会议管理系统是一个基于 WebRTC 技术的实时视频会议与直播平台，提供视频会议、在线直播、实时表决、权限管理等功能。系统采用前后端分离架构，支持多人实时音视频通信、桌面共享、在线投票等协作功能。

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| **后端框架** | Python + FastAPI (支持异步和 WebSocket) |
| **前端框架** | Vue 3 + Vite + Element Plus |
| **数据库** | SQLite (可无缝迁移至 MySQL) + SQLAlchemy ORM |
| **实时通信** | WebSocket + WebRTC (P2P Mesh 网络拓扑) |
| **鉴权机制** | JWT (JSON Web Token) |
| **部署方式** | Nginx 反向代理 + ngrok 内网穿透 (可选) |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      表现层 (Vue 3)                          │
│  UI渲染、用户交互、音视频流播放展示、WebRTC媒体处理           │
└─────────────────────────────────────────────────────────────┘
                              ↓ HTTP/WebSocket
┌─────────────────────────────────────────────────────────────┐
│               网关/路由层 (FastAPI Router)                   │
│  RESTful API、WebSocket长连接、请求路由分发                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              业务逻辑层 (Python Services)                    │
│  会议预约、权限校验、表决统计、WebSocket连接管理              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              数据层 (SQLAlchemy + SQLite)                    │
│  用户、会议、直播、表决、模板等数据持久化                     │
└─────────────────────────────────────────────────────────────┘
```

## 三、快速启动

### 3.1 启动方式

项目提供了统一的启动脚本 `start.sh`，支持多种启动模式：

```bash
# 同时启动前后端 (默认，推荐开发模式)
./start.sh

# 生产模式: Nginx + Ngrok 代理部署
./start.sh prod

# 仅启动后端 API 服务
./start.sh backend

# 仅启动前端 Vite 开发服务
./start.sh frontend

# 仅构建前端生产版本
./start.sh build

# 显示帮助
./start.sh help
```

### 3.2 端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8001 | FastAPI 服务 |
| 前端开发 | 5173 | Vite 开发服务器 |
| 生产入口 | 80 | Nginx 反向代理 |

### 3.3 访问地址

- **开发模式**:
  - 前端: http://127.0.0.1:5173
  - 后端: http://127.0.0.1:8001
  - API文档: http://127.0.0.1:8001/docs

- **生产模式**:
  - 通过 Nginx 80 端口统一访问
  - 可通过 ngrok 获取 HTTPS 公网地址

## 四、功能模块详解

### 4.1 用户权限模块

#### 4.1.1 角色定义

| 角色 | 权限范围 |
|------|----------|
| **admin (管理员)** | 全部权限：用户管理、会议模板管理、创建会议/直播、参与会议 |
| **host (主持人)** | 创建会议、发起直播、发起表决、参与会议 |
| **user (普通用户)** | 加入会议、观看直播、参与表决 |

#### 4.1.2 前端视图

| 路由 | 组件 | 功能 |
|------|------|------|
| `/login` | LoginView.vue | 用户登录 |
| `/register` | RegisterView.vue | 用户注册 |
| `/admin/users` | UserManageView.vue | 用户管理 (仅管理员) |

#### 4.1.3 登录页面 (LoginView.vue)

**路由**: `/login`
**权限**: 公开页面，无需登录

**页面布局**:
- 采用左右分栏布局（大屏设备）
- 左侧：品牌展示区，包含系统Logo、标题、系统简介和功能特性卡片
- 右侧：登录表单区，包含用户名输入、密码输入、登录按钮和注册链接

**功能说明**:
1. **用户名输入**: 3-50字符，支持中文、英文、数字
2. **密码输入**: 6-128字符，支持显示/隐藏密码
3. **登录操作**: 验证通过后获取JWT Token，自动跳转到仪表盘
4. **注册跳转**: 点击"立即注册"链接跳转到注册页面

**交互流程**:
```
输入用户名/密码 → 点击登录 → 调用登录API → 获取Token → 存储到Pinia → 跳转仪表盘
```

**错误处理**:
- 用户名或密码错误：显示"登录失败"提示
- 网络错误：显示网络错误提示

---

#### 4.1.4 注册页面 (RegisterView.vue)

**路由**: `/register`
**权限**: 公开页面，无需登录

**页面布局**:
- 与登录页面采用相同的左右分栏布局
- 左侧展示系统功能特性，右侧为注册表单

**功能说明**:
1. **用户名输入**: 首次注册的用户自动成为管理员
2. **密码输入**: 支持密码显示/隐藏切换
3. **注册操作**: 验证通过后自动登录并跳转到仪表盘
4. **登录跳转**: 已有账号可点击"返回登录"

**特殊规则**:
- 第一个注册的用户自动获得管理员(admin)角色
- 后续注册用户默认为普通用户(user)角色

### 4.2 仪表盘模块

#### 4.2.1 仪表盘页面 (DashboardView.vue)

**路由**: `/dashboard`
**权限**: 所有登录用户

**页面布局**:
- 左右两栏布局
- 左侧：日历组件，显示会议日期标记
- 右侧：当日会议议程列表

**功能说明**:

1. **日历视图**:
   - 使用Element Plus日历组件
   - 有会议的日期显示深色背景标记
   - 点击日期可查看当日会议议程

2. **会议议程**:
   - 显示选中日期的会议列表
   - 每个会议卡片包含：
     - 时间范围（开始时间-结束时间）
     - 会议标题
     - 状态标签（已预约/进行中/已结束）
     - "进入会议"快捷入口

3. **状态显示**:
   - 已预约（蓝色）：会议尚未开始
   - 进行中（绿色）：会议正在进行
   - 已结束（灰色）：会议已经结束

**交互流程**:
```
进入页面 → 加载所有会议数据 → 日历标记有会议日期 → 默认显示今日议程 → 点击日期切换议程 → 点击进入会议跳转会议室
```

### 4.3 视频会议模块

#### 4.2.1 功能特性

- **会议预约**: 创建会议、设置主题和时间
- **会议室**: 实时音视频通信、桌面共享
- **会议状态**: scheduled (待开始) / ongoing (进行中) / ended (已结束)
- **会议模板**: 快速创建预设配置的会议

#### 4.3.2 前端视图

| 路由 | 组件 | 功能 |
|------|------|------|
| `/meetings` | MeetingListView.vue | 会议列表、创建会议 |
| `/meetings/:id` | MeetingRoomView.vue | 会议室控制台 |

#### 4.3.3 会议列表页面 (MeetingListView.vue)

**路由**: `/meetings`
**权限**: 所有登录用户

**页面布局**:
1. **顶部区域**: 页面标题、描述、操作按钮（创建会议、从模板创建）
2. **统计卡片**: 四列展示会议统计数据
   - 全部会议：当前系统内可访问的会议总数
   - 进行中：正在使用会议室的场次（黄色）
   - 待开始：已排期、等待进入的会议（绿色）
   - 已结束：已完成的历史会议（红色）
3. **会议列表**: 表格形式展示所有会议
4. **创建对话框**: 普通创建和模板创建两种方式

**功能说明**:

1. **创建会议**（管理员/主持人）:
   - 点击"创建会议"按钮打开创建对话框
   - 填写会议主题、开始时间、结束时间
   - 提交后刷新会议列表

2. **从模板创建**（管理员/主持人）:
   - 点击"从模板创建"按钮
   - 选择启用的会议模板
   - 模板预览显示：名称、描述、默认标题、默认时长、容量、标签
   - 可覆盖模板的默认设置
   - 若模板设置了默认时长，结束时间可留空自动计算
   - 创建成功后自动跳转到会议室

3. **会议列表**:
   - 显示字段：主题、开始时间、结束时间、状态
   - 操作：点击"进入会议室"跳转到会议室

**权限控制**:
- 普通用户只能查看会议列表和进入会议室
- 管理员和主持人可以创建会议

---

#### 4.3.4 会议室页面 (MeetingRoomView.vue)

**路由**: `/meetings/:id`
**权限**: 所有登录用户

**页面布局**:
1. **顶部信息区**:
   - 左侧：会议室标题、描述
   - 右侧：房间号显示

2. **状态摘要栏**: 三列显示
   - 本地设备状态（已接入/未开启）
   - 桌面共享状态（共享中/未共享）
   - 在线成员数量

3. **主内容区**: 左右两栏布局
   - 左侧：视频画面网格（本地画面、桌面共享、远端流）
   - 右侧：侧边面板（聊天/文档/会议纪要/表决）

4. **底部控制栏**: 固定在底部，包含操作按钮

**核心功能**:

1. **本地设备控制**:
   - 打开/关闭摄像头和麦克风
   - 点击按钮切换设备状态
   - 设备开启后自动将流添加到所有WebRTC连接

2. **桌面共享**:
   - 点击"桌面共享"开启屏幕捕获
   - 支持选择共享整个屏幕、窗口或标签页
   - 共享结束自动停止推流
   - 支持手动停止共享

3. **WebRTC实时通信**:
   - 通过WebSocket建立信令通道
   - 使用STUN服务器进行NAT穿透
   - 支持P2P Mesh网络拓扑
   - 自动处理新用户加入、用户离开

4. **在线用户管理**:
   - 显示所有在线成员
   - 支持举手功能（显示举手标签）
   - 用户加入/离开实时更新

5. **在线表决**（管理员/主持人）:
   - 点击"发起表决"打开表决对话框
   - 填写表决主题和选项（默认：赞成/反对/弃权）
   - 发起后通过WebSocket广播给所有参会者
   - 实时显示投票结果和比例
   - 支持结束表决

6. **侧边面板**:
   - 聊天面板（开发中）
   - 文档面板（开发中）
   - 会议纪要面板（开发中）
   - 表决面板：显示当前表决状态和结果

**WebSocket消息类型**:

| 类型 | 方向 | 说明 |
|------|------|------|
| `join` | 客户端→服务端 | 加入房间，携带用户信息 |
| `leave` | 客户端→服务端 | 离开房间 |
| `user-list` | 服务端→客户端 | 在线用户列表 |
| `user-joined` | 服务端→客户端 | 新用户加入通知 |
| `user-left` | 服务端→客户端 | 用户离开通知 |
| `raise-hand` | 客户端→服务端 | 举手 |
| `lower-hand` | 客户端→服务端 | 放下举手 |
| `offer` | 客户端→服务端 | WebRTC Offer SDP |
| `answer` | 客户端→服务端 | WebRTC Answer SDP |
| `ice-candidate` | 客户端→服务端 | ICE Candidate |
| `vote-started` | 服务端→客户端 | 表决发起通知 |
| `vote-result` | 服务端→客户端 | 投票结果更新 |
| `vote-ended` | 服务端→客户端 | 表决结束通知 |
| `chat-message` | 客户端→服务端 | 聊天消息 |

**生命周期管理**:
- 进入页面：建立WebSocket连接、加载表决数据
- 离开页面：关闭设备、停止共享、关闭WebSocket、清理WebRTC连接

- **直播回看**: 播放录制的直播视频
- **实时聊天**: 观众与主播实时互动
- **举手互动**: 观众举手请求发言

#### 4.4.2 前端视图

| 路由 | 组件 | 功能 |

2. **直播列表**:
   - 显示字段：标题、房间号、开始时间、状态

   - 左侧：视频画面区域

     - 观众：显示主播画面

**核心功能**:

   - 点击"开始采集"获取摄像头和麦克风权限
   - 获取成功后显示本地预览画面
   - 自动向观众推流

2. **观众观看**:
   - 进入直播间自动接收主播画面
   - 支持实时聊天互动
   - 支持举手功能

3. **分会场管理**:
   - 支持主会场和多个分会场
   - 分会场之间可快速切换
   - 切换时自动断开当前连接并重新连接

4. **实时聊天**:
   - 发送文字消息
   - 发送图片（通过文件上传）
   - 消息实时广播给所有用户

5. **在线成员**:
   - 显示所有在线用户列表
   - 显示用户角色和举手状态
   - 主播可看到观众举手提醒

**身份区分**:

| 功能 | 主播（admin/host） | 观众（user） |
|------|-------------------|-------------|
| 开始采集 | ✓ | ✗ |
| 推流 | ✓ | ✗ |
| 观看直播 | ✓ | ✓ |
| 发送消息 | ✓ | ✓ |
| 举手 | ✓ | ✓ |

**WebRTC连接模式**:
- 主播：收到观众join消息后主动创建Offer
- 观众：被动接收主播的Offer并回复Answer
- 采用P2P Mesh模式，主播与每个观众建立独立连接

### 5.9 WebSocket 接口

#### WebSocket `/ws/meetings/{meeting_id}`
会议室信令通道

**连接参数**:
- URL: `ws://host/ws/meetings/{meeting_id}` 或 `wss://host/ws/meetings/{meeting_id}`
- meeting_id: 会议ID

**消息类型**:

| 类型 | 方向 | 说明 |
|------|------|------|
| `join` | 客户端 → 服务端 | 加入房间，携带用户信息 |
| `leave` | 客户端 → 服务端 | 离开房间 |
| `user-list` | 服务端 → 客户端 | 在线用户列表 |
| `user-joined` | 服务端 → 客户端 | 新用户加入通知 |
| `user-left` | 服务端 → 客户端 | 用户离开通知 |
| `raise-hand` | 客户端 → 服务端 | 举手请求 |
| `lower-hand` | 客户端 → 服务端 | 放下举手 |
| `chat-message` | 客户端 → 服务端 | 发送聊天消息 |
| `offer` | 客户端 → 服务端 | WebRTC Offer SDP |
| `answer` | 客户端 → 服务端 | WebRTC Answer SDP |
| `ice-candidate` | 客户端 → 服务端 | ICE Candidate |
| `vote-started` | 服务端 → 客户端 | 新表决发起通知 |
| `vote-result` | 服务端 → 客户端 | 投票结果实时更新 |
| `vote-ended` | 服务端 → 客户端 | 表决结束通知 |

**消息格式示例**:

```javascript
// 加入房间
{
  "type": "join",
  "from": "user-connection-id",
  "user_id": 1,
  "display_name": "张三",
  "role": "host"
}

// 聊天消息
{
  "type": "chat-message",
  "from": "user-connection-id",
  "msg_type": "text",  // text 或 image
  "content": "大家好",
  "url": null,  // 图片消息时填写
  "filename": null
}

// 举手
{
  "type": "raise-hand",
  "from": "user-connection-id"
}

// WebRTC Offer
{
  "type": "offer",
  "from": "user-connection-id",
  "to": "target-connection-id",
  "sdp": { ... }
}
```

---

#### WebSocket `/ws/live/{live_id}`
直播间信令通道

**连接参数**:
- URL: `ws://host/ws/live/{live_id}` 或 `wss://host/ws/live/{live_id}`

**消息类型**: 与会议室相同，支持以下功能：

- WebRTC信令（主播推流、观众拉流）

**主播与观众的连接模式**:
1. 观众发送 `join` 消息加入直播间
2. 主播收到消息后主动创建 `offer`
3. 观众收到 `offer` 后回复 `answer`
4. 双方交换 `ice-candidate` 建立连接

### 6.2 实体关系图

```
User ─────┬─────────── Meeting ─────── Vote
          │                │              │
          │                │              │
          │                ↓              ↓
          │            VoteOption    VoteRecord
          │
          ├────────── LiveStream (分会场parent_id指向主会场)
          │                │
          │                └── sub_venues (分会场列表)
          │
          ↓
      MeetingTemplate
```

**关系说明**:
- User 与 Meeting: 一对多（一个用户可创建多个会议）
- User 与 LiveStream: 一对多（一个用户可创建多个直播）
- Meeting 与 Vote: 一对多（一个会议可有多个表决）
- Vote 与 VoteOption: 一对多（一个表决可有多个选项）
- Vote 与 VoteRecord: 一对多（一个表决可有多条投票记录）
- LiveStream 与 LiveStream: 自关联（分会场parent_id指向主会场）
- User 与 MeetingTemplate: 一对多（一个用户可创建多个模板）

## 十、常见问题

### Q1: 摄像头/麦克风无法访问?

**原因**: WebRTC 需要 HTTPS 或 localhost 环境

**解决方案**:
- 本地开发使用 `http://127.0.0.1:5173`
- 生产环境使用 ngrok 获取 HTTPS 地址

---

### Q2: WebSocket 连接失败?

**检查项**:
1. 后端服务是否正常运行
2. Nginx WebSocket 代理配置是否正确
3. 浏览器是否支持 WebSocket

---

### Q3: 如何迁移到 MySQL?

**步骤**:
1. 修改 `config.py` 中的 `database_url`
2. 安装 MySQL驱动: `pip install mysqlclient`
3. 创建数据库
4. 重新运行应用(自动创建表结构)
